Understanding the links between landscape resources, climate stressors, and livelihoods.

Livelihood-Landscape links

To understand the links between landscape resources, livelihoods, climatic impacts to landscape resources, and changes to the functioning of key ecosystems we undertook the following activities:

  • Household survrey of coastal communities in Ba to estimate the value of ecosystem service flows from landscape resources and the sensitivity of these flows to climatic shocks.
  • Participatory modelling exercises in Fiji and Tonga to capture cause-effect-response relationships associated with problems people face in using landscape resources.
  • Semi-structured interviews with key landscape stakeholders at national and local levels.
  • Linking coastal / mangrove based livelihood activities with remotely sensed indicators of mangrove health.
  • Fieldwork in the Ba mangroves, Fiji, to identify patterns of carbon sequestration and spatial patterns of ecosystem condition.

The information gathered through these activities underpins the application development process and ensures that the applications provide tools for capturing and analysing how the landscape is used at different scales and by different people.

Use mobile GIS to map landscape features and document their attributes in the field.

QField for mobile data collection

We reviewed the functionality and characteristics of over 100 mobile GIS data collection apps against a set of technical criteria and criteria defined by stakeholders and end-user requirements. QField was identified as the most suitable app for capturing geospatial information about landscape use and condition in the field. QField is open-source, based on the QGIS desktop GIS platform, integrates easily with spatial databases, and allows for viewing secondary spatial layers in-the-field. In the field it provides touch-optimised map interface for digitising spatial features and digital forms for capturing attribute and non-spatial data.

Dashboards, servers, and databases to manage and analyse geospatial data.

System Architecture Diagram

We have developed server-side apps and spatial databases for managing geospatial data collected in-the-field using QField. Using the Shiny framework a dashboard has been created to enable syncing of data collected on multiple mobile devices, explore data in dynamic tables, web maps, and create charts and graphs.

  • Web forms to submit data collected on mobile devices.
  • Express.js server-side apps to handle submitted data and interface with secure cloud storage.
  • map.landscape - Shiny dashboard for data management, analysis, and interaction.

Use satellite images and machine learning to monitor inter-annual land cover dynamics within landscapes.

Inter-annual land cover monitoring

Using ESA Sentinel-2 images and ancillary datasets, a machine learning model to map inter-annual land cover dynamics at a 10 m spatial resolution has been developed. Initial field validation of this method has been undertaken in Ba, Fiji, in collaboration with the Ministry of Forests Forest Resource and Assessment Group (FRAC).
