In The Field

Map landscapes with mobile GIS.

QField for mobile data collection

We reviewed the functionality and characteristics of over 100 mobile GIS data collection apps against a set of technical criteria and criteria defined by stakeholders and end-user requirements. QField was identified as the most suitable app for capturing geospatial information about landscape use and condition in the field. QField is open-source, based on the QGIS desktop GIS platform, integrates easily with spatial databases, and allows for viewing secondary spatial layers in-the-field. In the field it provides touch-optimised map interface for digitising spatial features and digital forms for capturing attribute and non-spatial data.

In The Cloud

Server apps and databases to sync data collected on mobile devices.

Data management and visualisation

We have developed a server application and spatial databases for secure management geospatial data collected in-the-field using QField on multiple mobile devices. Using a simple deployment process, set up secure cloud storage and a web application to manage data collected in-the-field. Data collectors using QField can can submit their landscape data via a web form in their browser and the server app will screen this data and add it to secure cloud storage.

  • Web forms to submit data collected on mobile devices.
  • Express.js server-side app to handle submitted data and interface with secure cloud storage.

On Your Desktop

Use dashboards to analyse and visualise landscape data collected in-the-field.

Data management and visualisation

We developed a dashboard application to explore geospatial data using interactive tables, web maps, and charts. map.landscape is developed using the Shiny framework. It uses dplyr for data processing and sf for handling spatial data.

The motivation to develop map.landscape was to quickly explore and sync data collected using QField mobile GIS. However, it supports spatial and non-spatial data in GeoPackage files and is relevant for a range of data exploration and visualisation use cases.

  • Sync data collected using QField on different mobile devices.
  • Upload spatial and non-spatial tabular data as GeoPackage or zip files.
  • Explore data in interactive tables (using DataTables).
  • Combine tabular data using spatial and non-spatial joins.
  • Create summary tables.
  • Create and style web maps (using Leaflet).
  • Visualise data using different charts (using ggplot2).

At Scale

Use satellite images and machine learning to monitor inter-annual landscape dynamics.

Inter-annual land cover monitoring

Using ESA Sentinel-2 images and ancillary datasets, a machine learning model to map inter-annual land cover dynamics at a 10 m spatial resolution has been developed. Initial field validation of this method has been undertaken in Ba, Fiji, in collaboration with the Ministry of Forests Forest Resource and Assessment Group (FRAC).
